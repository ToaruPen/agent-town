# agent-town 設計仕様書

日付: 2026-07-22
ステータス: レビュー待ち

## 1. コンセプト

LLMエージェントの住民が自律的に働き、記憶し、関係を築きながら街を拡大していく2Dトップダウンの「人工生命水槽」ゲーム。Song of Syx / RimWorld系のコロニーシムの外見を持つが、住民の意思決定の中核はLLM（Smallville型）。プレイヤーは普段は観察者・対話者として関わり、必要に応じて神（資源配置・イベント発生）や市長（方針布告）として介入する。

- 初期住民: 3〜5体。上限: 10体程度
- ローカルマシンで動かす個人用ゲーム。公開Webデプロイは対象外

## 2. 技術スタック

three.jsは不採用。2Dトップダウンでは3Dレンダラの持ち味が出ず、本プロジェクトの主戦場は描画ではなくシミュレーションとLLMオーケストレーションであるため。

| 層 | 技術 | 備考 |
|---|---|---|
| フロントエンド | PixiJS + Vite + TypeScript | 状態を映すだけの薄い描画層 |
| バックエンド | Node.js + TypeScript、WebSocket (`ws`) | シミュレーションの権威サーバ |
| LLM実行 | `claude -p`（ヘッドレス）を子プロセス起動 | サブスクリプション枠で動作。JSON構造化出力 |
| 永続化 | JSONファイル（セーブデータ、住民ごとの記憶JSONL） | v1にDBは導入しない |
| テスト | Vitest | シム層は決定論的でユニットテスト向き |

構成原則: **バックエンドが唯一の真実（authoritative server）**。シミュレーションは固定ティック（10 tick/秒目安）でNode側で回し、ブラウザはWebSocket経由の状態を描画するだけ。ブラウザを閉じても街は育ち続ける。

## 3. 住民の頭脳: 2層アーキテクチャ

**プランナー（LLM・遅い・賢い）** と **エグゼキュータ（決定論的・速い・従順）** を分離する。

### プランナー
- `claude -p` を1回呼び、ペルソナ・記憶要約・街の現況を渡して「計画（タスク列）」をJSONで受け取る
- 呼び出しトリガー: (a) ゲーム内の朝 (b) 計画破綻時（資材切れ等） (c) プレイヤーとの対話時 (d) 重大イベント発生時
- 出力はJSONスキーマで検証し、実行可能性チェック（存在しない建物・座標等を拒否）を通す。失格時は再試行1回、それでも駄目ならルールベースの偽プランナーに退避

### エグゼキュータ
- 計画中のタスク（「木を10本伐る」「家を建てる」）をA*パスファインディング＋作業進行ルールで執行
- LLMは一切関与しない。完全に決定論的

### 思考スケジューラ
- CLI同時実行は最大2プロセス。住民ごとにクールダウンを設定
- 思考待ちの住民は前の計画を続行する（リアルタイム＋非同期思考）
- サブスクのレートリミット接近を検知したら偽プランナーへ自動退避し、街を止めない

## 4. 記憶システム（簡易Smallville）

- 住民ごとに記憶ストリーム（JSONL）。各記憶に重要度スコアを付与
- プランナー呼び出し時は recency＋重要度で記憶を選抜してプロンプトに載せる
- ゲーム内数日ごとに「振り返り」呼び出しで記憶を要約・凝縮（性格変化・人間関係の記憶の源泉）
- 埋め込みベクトル検索はv1では導入しない。recency＋重要度で面白さが立証されてから検討

## 5. 世界とゲームルール

- タイルグリッドマップ（森・水・平地・岩）
- 資源: 木材・食料・石の3種から開始
- 建物: 家・倉庫・畑・伐採所程度から開始。**何をどこに建てるかは住民のLLMが決める**（本作の見せ場）
- 住民パラメータ: 空腹・疲労。閾値超過で計画に割り込む（生存圧が物語を生む）
- 街が一定条件を満たすと新住民が移住してくる（拡大ループ）

## 6. プレイヤー介入

1. **観察**: 住民クリックで現在タスク・今日の計画・記憶の抜粋を閲覧
2. **対話**: 住民とチャット。会話内容は住民の記憶に書き込まれ、以後の行動に影響
3. **市長（布告）**: 全住民の次回プランニングのコンテキストに載る方針を布告。従うか曲解するかは住民次第
4. **神の手**: 資源の配置、イベント（嵐・来訪者）の発生

## 7. マイルストーン

| # | 名称 | 内容 | 完了条件 |
|---|---|---|---|
| M1 | 動く水槽（LLMなし） | マップ・ティック・A*移動・偽プランナーで住民が働く | ブラウザで住民が資源を採集して運ぶ様子が見える。シム層のテスト整備 |
| M2 | 一人目の知性 | 住民1体のプランナーを`claude -p`に差し替え | LLM計画で1日が回る。検証・フォールバック動作確認 |
| M3 | 社会と拡大 | 複数住民・記憶ストリーム・振り返り・建設判断・移住 | 住民が自発的に建物を建て、街が拡大する |
| M4 | プレイヤー登場 | 対話UI・布告・神の手 | 対話が記憶に残り行動に影響することを確認 |
| M5 | 磨き | セーブ/ロード・速度調整・観察UI充実 | 中断・再開して継続プレイできる |

各マイルストーンは「遊べる状態」で終わること。M1はLLMゼロで完結し、シム設計の失敗を安く発見する。

## 8. リスクと対策

- **レートリミット枯渇**: 思考スケジューラで頻度制御。偽プランナー自動退避で街は止まらない
- **LLM出力の破綻**: スキーマ検証＋実行可能性チェック → 再試行1回 → 偽プランナー
- **「眺めて面白い」の不成立**: M1時点でゲームループの手応えを確認してからLLMに投資する

## 9. リポジトリ構成・ツーリング

- **pnpm workspace のモノレポ**: `packages/server`（シム＋LLMオーケストレーション）、`packages/client`（PixiJS描画）、`packages/shared`（型・プロトコル定義）
- **タスクランナー**: `just`（`just dev` / `just test` / `just check` 等）
- **品質ゲート（AIスロップ防止設定）**:
  - TypeScript `strict` 全有効＋`noUncheckedIndexedAccess`
  - Biome（lint＋format統合）: `noExplicitAny`、`noExcessiveCognitiveComplexity`（上限10）、未使用コード検出をerror級で
  - エラー握りつぶし禁止（空catch・握りつぶしlintで検出）
  - Vitest。シム層はロジックが決定論的なのでテストを厚く
- **アセット**: Kenney.nl等のCC0素材を第一候補、不足分はCodexのimagegenで生成
- **エージェント向け文書**: リポジトリ直下に `AGENTS.md`（英語）を置き、`CLAUDE.md` はそれを参照する。設計判断は `docs/` 配下に集約

## 10. 開発体制

- **監督官: Claude Code（本セッション）** — 設計・タスク分割・受け入れ検証・進行管理
- **作業員: codex delegate**（`~/.claude/commands/codex/scripts/codex-delegate.sh`）— 実装タスクの実行
- **レビュワー: codex review**（`codex-review.sh`）— マイルストーン単位の差分レビュー
- 監督官側のレートリミット接近時は追加のcodexエージェントへ作業を委譲し、ユーザーがchatgpt.app側でセッションを確認できるように通知する
- codexへのプロンプトは簡潔に（<1KB、ゴールと参照ディレクトリのみ。コード貼り付け禁止）
- コミットメッセージ・PR・ドキュメントにローカル絶対パス（/Users/...）を含めない
