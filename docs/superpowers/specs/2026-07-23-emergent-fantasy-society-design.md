# 創発的ファンタジー社会シミュレーション設計仕様書

日付: 2026-07-23
ステータス: レビュー待ち

## 1. 目的

agent-townを、住民が資源を集めるだけの水槽から、**世界の来歴・外圧・住民の欲望が集団と制度を生み、その制度が都市の形になる社会シミュレーション**へ発展させる。

プレイヤーが眺めるのは、あらかじめ置かれた「それらしい街」ではない。次の因果が地図上で読み取れる街である。

> 古い戦争で故郷を失った入植者が、凶作への恐怖から備蓄を求める
>
> → 備蓄を支持する集団が生まれる
>
> → 集団が穀物評議会として制度化する
>
> → 労働力と木材を集めて共同穀倉を建てる
>
> → 穀倉の周囲に市場と道路が育つ

建物、区画、役職、儀礼、対立には「なぜ存在するか」を説明できる来歴を持たせる。見栄えは結果であり、原因の代用品にはしない。

## 2. 既存仕様との関係

本仕様は `2026-07-22-agent-town-design.md` を土台にした拡張・方針変更である。矛盾する場合は、次の項目について本仕様を優先する。

- 初期住民数と最終的な人口規模
- 全住民の意思決定にLLMを使う方針
- 固定的な建物アンロックを中心にした街の拡大
- 世界に来歴がなく、現在の入植地だけをシミュレートする前提
- プレイヤーを直接的な建設者として扱う前提

複数LLMプロバイダーの実行・隔離・ルーティング契約は `2026-07-22-multi-provider-llm-routing-design.md` を維持する。ただし、ゲーム内の通常判断は決定論的なシミュレーションが担い、Codexを既定の高価値思考プロバイダー、Claudeを明示割当時だけ使うプロバイダーとする。現在の実装がこの既定値に追随しているかは、実装計画の開始時に確認する。

## 3. 体験の柱

### 3.1 都市は社会の可視化である

建築物はメニューから直接アンロックされない。住民の需要、集団の支持、制度の権限、資源と場所の制約がそろったときに建設提案となる。

同じ「食料不足」でも、国柄と経験によって結果は変わる。

- 相互扶助を重んじる共同体は共同穀倉を作る
- 商業を重んじる共同体は穀物市場と信用制度を作る
- 軍事的な共同体は配給所と倉庫番を置く
- 信仰の強い共同体は祭祀と供物によって不安へ応じる

どれも固定ルートではない。状況、人物、過去の成功と失敗によって別の形へ変化できる。

### 3.2 外圧は欲望を社会問題へ変える

住民は平時にも欲望を持つが、制度形成を強く促すのは外圧である。

- 季節、寒波、干ばつ、病害
- 食料、住居、治安、労働力の不足
- 周辺国家との交易、圧力、移民、戦争
- 遺跡、禁忌、魔術的異常
- 身分、信仰、知識、富をめぐる内部対立

外圧は単なるランダム罰ではない。地理、過去の出来事、周辺勢力の行動から発生し、住民が予測・記憶・対策できる。

### 3.3 国柄は初期条件であり運命ではない

国家や移民集団には国柄を与える。国柄は全住民を同じ性格にするラベルではなく、価値観と制度選好の確率分布である。

新しい入植地は出身国の文化を継ぐが、新天地の環境、世代交代、混住、危機への対応によって変化する。プレイヤーは「勤勉国だから工業化する」といった決め打ちではなく、継承と変容を観察する。

### 3.4 ローファンタジーは因果へ参加する

魔法は稀だが実在し、世界の歴史と社会制度へ長い影を落とす。

- 魔力は地理的に偏在し、湧出地・遺物・血統・儀礼などの媒介を必要とする
- 利用には健康、時間、希少資源、社会的信頼、環境汚染などの代償がある
- 効果は一貫した法則に従い、都合のよい万能解決にならない
- 多くの住民は魔法を直接使わないが、伝承、禁忌、権威、交易品として影響を受ける
- 真偽不明の伝承と、シミュレーション上実在する現象を区別する

最初の実装では派手な呪文経済を作らない。ひとつの魔術的異常が歴史上の移住や禁忌、現在の資源利用へつながることを示す。

### 3.5 プレイヤーは観察者であり、間接的な介入者である

プレイヤーは建物を直接置く市長ではない。住民と制度へ働きかける。

- 情報を見る: 人物、集団、制度、建物、出来事の因果を調べる
- 語りかける: 個人や代表者との会話により認識や優先度へ影響する
- 布告する: 共同体全体へ価値や目標を提案する
- 兆しを与える: 資源、天候、夢、異常のような世界内の手段で誘導する

介入は命令の即時実行ではない。支持、反発、曲解、便乗が起こり得る。

## 4. シミュレーションの層

世界を次の五層に分ける。上位層は下位層から理由を受け取り、下位層へ結果を残す。

| 層 | 主な状態 | 主な変化 |
|---|---|---|
| 物理世界 | 地形、水、気候、土壌、生態、資源、魔力異常 | 季節、枯渇、災害、環境変化 |
| 歴史世界 | 国家、人口、交易路、戦争、宗教、遺跡、重大事件 | 興亡、移住、同盟、分裂、発見 |
| 共同体 | 国柄、人口構成、備蓄、評判、集団、制度 | 合意、対立、制度化、法や慣習 |
| 生活者 | 欲望、技能、関係、信念、役割、記憶 | 労働、会話、支持、離反、移住 |
| 空間 | 建物、道、区画、境界、利用頻度 | 建設、増築、転用、放棄、集積 |

サーバーがすべての権威状態を持ち、クライアントはその状態と来歴を描画する。シミュレーションの機械的結果はseed付き乱数を含む決定論的処理とし、LLMの自由文を直接ゲームルールとして実行しない。

## 5. 開始前の世界史生成

### 5.1 生成単位

新規ゲーム作成時に、現在から遡る200〜400年を集約モデルで事前シミュレートする。最初の可視スライスでは固定200年、4国家を推奨する。

1年または1季を1ステップとし、個々の住民ではなく国家・地域・人口集団を単位に処理する。

- 国家の成立、分裂、滅亡
- 人口の増減と移住
- 食料・素材・希少品の生産と交易
- 国境紛争、侵攻、同盟
- 宗教運動、改革、禁忌
- 災害、疫病、魔術的異常
- 都市、街道、砦、聖地、遺跡の成立と放棄

生成結果は現在の地図に物証として残す。歴史文だけを生成して終わらせない。

### 5.2 歴史因果

各歴史イベントは最低限、原因・主体・対象・結果を持つ。

```ts
interface WorldEvent {
  id: EventId;
  year: number;
  kind: WorldEventKind;
  actorIds: EntityId[];
  subjectIds: EntityId[];
  causeIds: EventId[];
  effects: WorldEffect[];
  visibility: "known" | "rumored" | "hidden";
}
```

`causeIds`は説明用の飾りではなく、実際にイベントを成立させた入力を指す。原因がない初期条件は、世界生成seedと地理状態へたどれるようにする。

### 5.3 入植地の開始

プレイは古い世界の辺境へ到着した新規入植団から始まる。

- 出身国家と、必要なら少数の別文化出身者を選ぶ
- 移住理由は世界史から導く。追放、土地不足、交易拠点、信仰、災害避難など
- 入植者は出身地の価値観、技能、傷、対外認識を不均一に継承する
- 開始地点には地理的利点と不利、周辺勢力、ひとつ以上の歴史的痕跡がある

世界史生成で現在が一意に決まり、その状態からライブシミュレーションを開始する。同じseedと設定なら同じ開始世界になる。

## 6. 国家と国柄

### 6.1 国柄の構造

国家は名前や色だけでなく、次を持つ。

```ts
interface PolityCharacter {
  valueWeights: Record<CulturalValue, number>;
  foundingMyth: BeliefId;
  formativeTraumaIds: EventId[];
  tabooIds: BeliefId[];
  ambitions: PolityAmbition[];
  governanceHabits: GovernanceHabit[];
  internalFactions: FactionProfile[];
}
```

初期の `CulturalValue` 候補は相互扶助、自由取引、武勇、秩序、信仰、知識、血縁、自然との調和とする。値は排他的なクラスではなく重みで表す。一国の中にも反対派や地域差がある。

### 6.2 形成と変化

国柄の一部は世界生成時の地理と創設事情から生まれ、一部は歴史イベントへの反応として変化する。

- 長い包囲を生き延びた国家は備蓄と防壁を重んじやすい
- 安定した交易路で栄えた国家は契約と異文化受容を重んじやすい
- 魔術災害を経験した国家は異常を禁忌にするか、逆に統制権力へ組み込む

変化には根拠イベントを残す。国柄画面では現在値だけでなく、「いつ、何によって傾向が強まったか」を表示する。

### 6.3 入植地への継承

入植団は出身国の国柄をそのままコピーしない。各入植者が異なる強さで価値観を持ち込み、共同体の文化は人口構成と経験から集約する。

これにより、出身国へ忠実な植民地、出身国への反発から生まれた新文化、複数文化が混ざった境界社会のいずれも同じ仕組みから成立する。

## 7. 欲望から制度への創発

### 7.1 個人の欲望

通常住民は最低限、次の欲望を0〜1の強度で持つ。

- 生存: 飢え、休息、安全、住居
- 繁栄: よりよい道具、財、安定した生活
- 地位: 承認、権限、熟達
- 所属: 家族、仲間、共同体
- 信仰: 世界の意味、儀礼、禁忌
- 知識: 探索、記録、技術
- 力: 自立、支配、防衛

強度は性格だけで決まらない。現在の不足、予測される危機、記憶、他者との関係、文化価値、所属集団によって変化する。

### 7.2 問題認識と提案

住民は欲望と観測から問題を認識し、解決案を支持または提案する。

```text
観測された状態
  + 欲望の不足
  + 記憶された出来事
  + 文化的な解釈
  = 問題認識

問題認識
  + 利用可能な知識
  + 資源と権限
  = 解決提案
```

最初の可視スライスでは食料不安を中心に、共同備蓄、私的取引、配給統制など複数の解決案を候補にする。候補は固定技術ツリーではなく、必要条件と文化的な選好を持つ制度原型として定義する。

### 7.3 集団

同じ問題認識や利益を一定期間共有する住民が集団を作る。

- 集団には支持者、代表者、目的、結束度、反対対象がある
- 支持は一人一派に限定しない
- 危機が去れば解散し、成功すれば存続し、対立すれば分裂できる
- 集団名や標語は表現であり、機械的性質は目的と構成員から決まる

### 7.4 制度

集団が十分な支持、資源、継続性を得ると制度化を提案する。

制度は最低限、次を持つ。

- 解決しようとする問題
- 支持者と反対者
- 役職と権限
- 維持費、必要労働、必要な場所
- 資源の収集・配分ルール
- 正統性の根拠
- 設立イベントと、その原因

制度が成立すると、役職の競争、税や供出、恩恵と不利益が生まれる。制度は必ずしも善ではなく、目的を失ったまま自己保存したり、権力を拡大したり、廃止されたりする。

### 7.5 建築と都市形態

制度や反復行動が空間要求を生む。

- 共同備蓄制度 → 穀倉、荷下ろし場、監視役、集散路
- 交易集団 → 市場、露店、宿、広場
- 防衛組織 → 見張り台、柵、兵舎、門
- 信仰集団 → 祠、墓地、巡礼路、禁足地
- 知識集団 → 記録所、工房、学校

建物の候補地は資源距離、移動頻度、防衛、所有、禁忌、既存施設との関係から評価する。日常的に踏まれた経路は小道となり、交通量と制度投資によって道路へ成長する。建物の集積が結果として地区を形成する。

## 8. 因果と履歴の共通モデル

### 8.1 根拠の保存

重要な状態変化には、説明可能な根拠を保存する。

```ts
interface Provenance {
  causedByEventIds: EventId[];
  proposedByAgentIds: AgentId[];
  supportedByAgentIds: AgentId[];
  opposedByAgentIds: AgentId[];
  decidedAtTick: number;
}
```

建物、集団、制度、文化変化は `Provenance` を参照する。UIはこの参照をたどり、「現在の状態 → 設立判断 → 当時の問題 → 世界史上の原因」を段階的に開ける。

### 8.2 決定と叙述の分離

ゲーム結果を決める構造化された事実と、読みやすい文章を分離する。

- 決定論的シミュレーション: 状態、候補、支持、投票、資源消費、成否
- LLM: 制度名、演説、年代記、代表者の言葉、重大判断での選好表明

LLMが「穀倉が完成した」と書いても、資源と作業が完了していなければ事実にはならない。叙述は必ず既存のイベントIDを根拠に生成する。

### 8.3 記憶の段階

すべての住民に無制限の自然言語記憶を持たせない。

- 通常住民: 重要イベントID、感情評価、関係変化を構造化して保持
- 代表者: 重要イベントの短い要約と将来目標を追加
- 共同体: 公的年代記、噂、禁じられた記録を分けて保持

これにより人口が増えても、因果を失わずLLM利用量を抑える。

## 9. LLMの役割と予算

### 9.1 利用する場面

Codexを既定プロバイダーとして、次の高価値な節目に限定して使う。

- 指導者や代表者が複数の実行可能な提案から選好を表明する
- 新しい集団、制度、条約、年代記に世界観と来歴に沿った名前を付ける
- 重大事件への演説、対話、噂、年代記を生成する
- プレイヤーとの会話を構造化された態度変化へ変換する

Claudeは住民または役割へ明示的に割り当てたときだけ使う。プロバイダー間の暗黙フォールバックは行わず、失敗時は決定論的な選択・定型文でシミュレーションを継続する。

### 9.2 利用しない場面

- 毎ティックの移動、採集、建設、食事、休息
- 全住民の日常判断
- 数値計算、ルール判定、投票集計
- 歴史シミュレーションの成否判定
- 建物の完成や資源の生成

### 9.3 呼び出し境界

LLMへ送るのは、公開可能なゲーム状態の最小限の要約と選択可能な候補だけとする。LLMは任意の座標、資源、イベントを新設できない。出力は既知IDと列挙値を持つ厳格なスキーマで検証し、不正な出力は状態へ適用しない。

最初の可視スライスでは、LLM停止中でも同じ因果ループを最後まで再現できることを必須とする。LLM品質は表現と人物らしさを高めるが、ゲーム成立の前提にはしない。

## 10. 可視化と操作

### 10.1 地図

通常表示だけで、最低限次が動いて見えるようにする。

- 住民が資源と施設の間を移動し、反復経路が小道になる
- 建設予定地に資材が集まり、段階的に建物が完成する
- 集団または制度に属する住民と施設の関係が識別できる
- 危機、提案、成立、建設の節目が画面上の短い通知として出る

常時表示する情報を増やしすぎず、詳細は選択時に開く。

### 10.2 因果インスペクター

住民、集団、制度、建物、国家をクリックすると、共通形式の因果カードを表示する。

```text
共同穀倉
目的: 冬の食料不足を防ぐ
設立: 12年 秋
提案: リラ
支持: 7 / 反対: 2
直接原因: 前冬の飢餓、備蓄不足
遠因: 故郷での包囲戦、入植団の相互扶助観
現在の効果: 食料腐敗 -15%、冬備蓄 +84
```

原因項目を選ぶと、関連する人物、場所、出来事へ移動できる。

### 10.3 世界と国柄

- 地域地図: 現存国家、交易路、遺跡、入植地との関係
- 年代記: 主要事件を年表で表示し、現在へ関係する事件を強調
- 国柄カード: 価値観、創設神話、傷、禁忌、野心、内部対立
- 出身と変容: 入植地が出身国から何を継ぎ、何を変えたかを表示

### 10.4 間接介入

最初の可視スライスでは観察を主とし、ひとつの軽い介入だけを提供する。候補は「共同体へ関心事を示す兆し」で、食料、安全、交易、信仰などの問題認識へ一時的な重みを加える。結果を保証せず、誰がどう解釈したかを履歴に残す。

## 11. 最初の可視vertical slice

### 11.1 目的

短い観察時間で、**世界史 → 国柄 → 個人の不安 → 集団 → 制度 → 建設 → 都市の形**が一続きの因果として見えることを証明する。

### 11.2 スコープ

- 同じseedで再現可能な200年・4国家の集約世界史
- 各国家の国柄カードと、現在に残る交易路または遺跡
- 1国家を主な出身地とする入植団
- 現行人数で開始し、既存の移住ループを維持する
- 食料と冬を最初の主要外圧にする
- 食料不安に対する複数の制度候補
- 支持者が集団を作り、ひとつの制度を成立させる
- 制度が共同穀倉などひとつの新規施設を要求する
- 資材運搬、建設進捗、完成、利用が地図上で見える
- 反復移動から少なくとも小道の可視状態が生まれる
- 世界年表、国柄カード、因果インスペクター
- ひとつの間接介入と、その解釈履歴
- LLM無効時にも全工程が完走する

### 11.3 明示的に後回しにするもの

- 数百〜数千人の個体シミュレーション
- 複数都市を同時にライブ実行する広域経済
- 完全な外交・戦争・世代交代
- 自由形式の宗教、法体系、技術発明
- 常時LLM会話する全住民
- 大量の建物種類と装飾アセット
- 万能魔法、戦闘呪文、魔法産業

最初の因果ループを成立させた後、通常住民を決定論的に増やして50〜100人規模を目指す。LLM管理者数は人口と独立させ、代表者・指導者・重要人物だけに制限する。

## 12. 実装境界

### 12.1 shared

- 世界史、国家、文化価値、集団、制度、因果参照の共有型
- クライアントへ送る読み取り用スナップショット
- ゲーム定数と列挙値

### 12.2 server/sim

- seed付き世界史生成
- 欲望と問題認識の更新
- 支持の集約、集団形成、制度成立
- 制度が要求する建設提案
- 建設と小道形成
- 因果イベントログ

すべて純粋かつ決定論的に保ち、`net/`、I/O、時刻API、LLMランナーを参照しない。

### 12.3 serverのオーケストレーション

- 決定済み候補に対するLLM選好・命名・叙述
- 既存の複数プロバイダー境界と呼び出し予算
- 検証済み応答をシミュレーションコマンドへ変換

### 12.4 client

- 地図上の施設、建設進捗、小道、所属の描画
- 世界年表、国柄カード、因果インスペクター
- 間接介入の入力

クライアントは制度成立や建設候補を決定しない。

## 13. テスト戦略

すべてRed → Green → Refactorで進める。

1. 世界史再現性: 同一seedで同じ国家・事件・現在状態、別seedで有意な差。
2. 歴史整合性: 参照IDが存在し、滅亡後の国家が主体にならず、現在の遺跡が成立・放棄イベントへたどれる。
3. 国柄形成: 地理と歴史イベントが値へ反映され、変化に根拠イベントがある。
4. 欲望更新: 不足、記憶、文化、外圧から期待した問題認識が生じる。
5. 集団形成: 支持閾値、継続期間、解散、分裂を決定論的に再現できる。
6. 制度成立: 必要な支持・資源・権限を欠く場合は成立せず、成立時は因果参照を持つ。
7. 空間結果: 制度の要求から建設提案が生まれ、資材と作業なしに完成しない。
8. 小道形成: 同じ経路の利用が蓄積し、閾値で表示状態が変わる。
9. LLM境界: LLMなしで進行し、不正IDや不可能な判断を拒否する。
10. 通信: welcome/updateで新しい状態が欠落せず、クライアントが権威状態だけを描画する。
11. UI: 主要な因果カードと世界年表をブラウザテストで確認する。

## 14. 受け入れ条件

最初の可視vertical sliceは、次をすべて満たしたとき完了とする。

- `just check` と `just test` が通る
- 同じseedの2回実行で、世界史と制度成立までの状態が一致する
- ブラウザを開いてから短い観察時間内に、食料不安から制度提案へ至る兆候が見える
- 少なくとも2つの制度候補に支持と反対が分かれ、ひとつが成立する
- 成立制度が要求した施設を、住民が資材運搬と作業によって完成させる
- 地図上に建物と小道という恒久的な空間結果が残る
- 施設を選ぶと、設立者、支持者、反対者、直接原因、世界史上の遠因を確認できる
- 国家を選ぶと、国柄とその形成根拠を確認できる
- LLMを無効化しても上記ループが停止しない
- LLMを有効化した場合、通常住民の日常行動では呼び出されず、指定した節目だけでCodexが使われる

## 15. 採用しない案

### 固定アンロック中心の街づくり

実装は容易だが、同じ条件から毎回同じ街になり、「なぜこの社会になったか」を弱める。制度原型は使うが、出現は欲望と支持の因果から導く。

### 全住民を常時LLMで動かす

費用、待ち時間、再現性、検証可能性が人口とともに悪化する。日常生活は決定論的に動かし、LLMは代表者と節目の表現へ限定する。

### LLMに世界史を一括執筆させる

読み物としては豊かでも、現在の資源、遺跡、国柄と機械的につながらないハリボテになりやすい。歴史の事実をシミュレーションし、LLMは事実に基づく年代記だけを書く。

### 高魔法世界

魔法が日常の万能技術になると、食料、物流、地理、制度の圧力を容易に無効化する。最初は稀少で代償のある異常として扱い、社会の反応を見せる。

## 16. 次の設計課題

本仕様の承認後、最初の可視vertical sliceを次の順で実装計画へ分解する。

1. 世界史・国家・国柄の決定論的な最小モデル
2. 現在世界スナップショットと年表・国柄表示
3. 食料不安から集団・制度が成立する因果ループ
4. 制度要求から穀倉建設へ至る空間ループ
5. 小道と因果インスペクター
6. 間接介入
7. Codexによる節目の命名・叙述

各段階を単独でブラウザから確認できる状態にし、見栄えだけが先行して因果が欠けないよう、シミュレーション状態と表示を同じ縦切りで進める。
